MapWindow Open Source GIS Lite v4.9

Thank you for using MapWindow GIS Lite v4.9
This version is a Lite version of MapWindow v4.8.8, but uses MapWinGIS v4.9 (the ActiveX control).

MapWindow v5

In 2015 we would like to start with the development of MapWindow v5.
This version will be a complete rewritten version of the .NET part of MapWindow. The lasts years we've investigated a lot of time and effort in optimizing the C++ part of MapWindow: MapWinGIS and it is now time to start with the overhaul of the .NET part.

Donations

To be able to start with the development we need some funding. With just $6,000 we can build the first beta of MapWindow v5, which will include the new architecture, the features of MapWindow v4.9 (PostGIS support, Shapefile Editor, grid proxy files, GDALv2.0) and at least 1 plug-in and 1 tool for the toolkit.
Assuming enough funding will be donated before the end of Januari 2015, we can have the first beta of MapWindow v5 ready for downloading late May 2015.

Wish list

Mid Januari 2015 this list will be up for voting. The list below is still unordered.
  • Loose coupling of MapWindow and MapWinGIS using a wrapper
  • Enhancing new wrapper with IEnumerable, IDisposable interfaces, hiding obsolete members, supplying some additional logic - things useful for plugin writers
  • Legend control, like in MWv4.8.8
  • Projections support, like in MWv4.8.8, including easy searching on EPSG-code
  • Tiles support, like in MWv4.8.8
  • Symbology support, like in MWv4.8.8
  • Enhance Symbology GUI, for raster layers
  • Print/lay-out control, like in MWv4.8.8
  • Add GUI for opening images and grids
  • Add GUI for opening shapefiles
  • Add a fast table editor that can handle large data. Perhaps use Syncfusion's Grid control
  • Layer class to communicate with map control
  • Data repository/browser for files, databases and servers
  • Make the code ReSharper and StyleCop compliant
  • Investigate if we can make the ocx registry-free using a manifest file as mentioned here.
  • Investigate if we can make a ClickOnce deployment with automated updates, releated to making the ocx registry-free.
  • Investigate how to best support the new GeoPackage format, which is already supported bij GDAL/OGR.
  • Add logging like Log4Net or NLog
  • Investigate if we can/should use the GeoAPI library
  • Use the Managed Extensibility Framework for the plug-ins and tools of the toolbox
  • Use the new features of .NET framework, like the async/await design pattern, dynamic variables and such.
  • Make tools for the toolbox of most GDAL/OGR options
  • Make tools for the toolbox of most GEOS options
  • Support WFS – Web Feature Service: for retrieving or altering feature descriptions
  • Support WMS – Web Map Service: provides map images
  • Support WCS – Web Coverage Service: provides access, subsetting, and processing on coverage objects
  • Support WMTS – Web Map Tile Service: provides map image tiles
  • Support CSW – Catalog Service for the Web: access to catalog information
  • Implement gdal_grid.exe
  • Support the GDAL GeoPackage raster format
  • Show usefull tips on start-up, like QGis does
